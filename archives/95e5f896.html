将Hexo博客备份到Github

发表于 2020-05-14 归档于 遇见hexo ， Github 阅读次数： 阅读次数：
本文字数： 1.7k 阅读时长 ≈ 2 分钟

前段时间折腾了下Hexo的next 5.0+和next 7.8+的版本，具体体验上很不错，由于自己的原因，老是忘记备份，放在文件夹又不见得回去把它翻出来，辛辛苦苦写的文章不能就这样没了，于是想到Hexo博客可不可以备份到Github呢？

一.使用hexo-git-backup插件

hexo-git-backup插件在Hexo官网的插件里是搜不到的，这个插件发布在了GitHub上，关于安装使用官方也给了很简洁易懂的说明文档。

安装插件

如果你的Hexo版本是2.x.x（查看Hexo版本可使用命令：hexo version）在终端中使用如下命令安装：
$ npm install hexo-git-backup@0.0.91 --save
如果你的Hexo版本是3.x.x+则使用如下命令安装:
$ npm install hexo-git-backup --save

插件的更新

如果是通过$ npm install hexo-git-backup --save命令安装的那么先要删除掉旧版本再升级，具体命令如下：
$ npm remove hexo-git-backup
$ npm install hexo-git-backup --save

配置

在hexo的_config.yml里配置这个插件：

   type: git
   theme: next,landscape
   message: Back up my blog
   repository:
      github: git@github.com:xxx/xxx.git,branchName
      gitcafe: git@github.com:xxx/xxx.git,branchName

详细参数解释：

  • theme：你要备份的主题名称 theme name,theme name

  • message：自定义提交信息

  • repository：仓库名，注意仓库地址后面要添加一个分支名，比如我就创建了一个Hexo backup分支

使用

从现在开始使用以下命令你就可以备份整个博客了！
$ hexo backup
或者使用以下简写命令也可以：
$ hexo b

注意

如果你按照上面的备份步骤备份了主题，那目录中的themes/theme name/.git这个文件就会被移除掉哦.

二.在Github.io上建立博客新分支

创建博客仓库新的分支

在Github.io上建立博客时已经开了一个新仓库了，如果再开另一个仓库存放源代码有点浪费，因此采用建立新分支的方法备份博客。不会建分支可以参考Git 分支 - 分支的新建与合并.博客都已经建立好了，那什么时候创建备份分支都是可以的.

针对博客已经搭建并发布过文章的

  1. 在你的博客仓库创建一个分支Hexo（这个命名随意，我这里以Hexo为参考）.

  2. 设置Hexo为默认分支（不知道怎么设的可以百度）.

将博客仓库clone至本地

将之前的Hexo文件夹中的:_config.yml,themes/,source,scffolds/,package.json,.gitignore复制到你克隆下来的仓库文件夹，即Username.github.io.（Username是你自己的用户名）

删除.git

将themes/next/(我用的是NexT主题)中的.git删除，否则后面无法将主题文件夹push.

运行命令

  1. 在Username.github.io文件夹下面执行npm install，npm install hexo-deployer-git(这里可以看看分支是不是显示为Hexo)。
  2. 同界面执行git add .、git commit -m ""、git push origin hexo来提交hexo网站源文件。
  3. 执行hexo g -d生成静态网页部署至Github上

这样一来，Username.github.io仓库就有master分支和hexo分支，分别保存静态网页和源文.

后续运行

在本地对博客修改（包括修改主题样式、发布新文章等）后：

  1. 依次执行git add .、git commit -m ""、git push origin hexo来提交hexo网站源文件.
  2. 执行hexo g -d生成静态网页部署至Github上.
    即重复运行命令的2-3步骤，以上两步没有严格的顺序.
